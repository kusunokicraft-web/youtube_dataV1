# タイトル設計ガイドライン

新規動画のタイトル提案時に参照するルール集。
チャネル内データ（n=28）と業界調査（VidIQ 128M 動画）を統合。

---

## 1. 確度の高いルール（信頼してよい）

### 強く推奨

| 要素 | 効果 | 根拠 |
|---|---|---|
| **「?」「!」 句読点** | CTR +0.67pp | チャネル内 n=8 |
| **「②」「③」 続編マーカー** | CTR +0.97pp | チャネル内 n=3、シリーズ施策と整合 |
| **【】ブラケット 1〜2 個** | 標準形式、無くしてはダメ | チャネル既存スタイル |
| **「注意」「爆笑」など感情ワード** | CTR +0.68pp | チャネル内 n=5 |

### 強く非推奨

| 要素 | 効果 | 根拠 |
|---|---|---|
| **長尺動画での絵文字（🔥💀🎉等）** | **視聴回数 -11%** | **VidIQ 128M 動画調査** |
| 60 字超のタイトル | サムネ上で truncate される | YouTube UI |
| 「まとめ」を 2 回以上使う | 冗長 | UX |

> **注意**: 元の分析で「絵文字 +1.09pp」と書いたが、これは n=2 の小サンプルでの誤誘導。
> **大規模データでは長尺動画の絵文字は視聴回数を 11% 下げる**ため、原則使わない。

| 要素 | 観測効果 | 信頼度 |
|---|---|---|
| 「劇場版」 | CTR -1.01pp | 弱（n=3、露出量交絡の可能性） |
| 「初コラボ」 | CTR -0.81pp | 中（n=10） |
| 「①」マーカー | CTR -0.80pp（②比） | 中（n=13） |

| 「総集編」「総まとめ」 | CTR -0.56pp | 中（n=19、ジャンル特性かも） |

## 3. 形式

```
【フック】+ [キャラ名 / 関係性 + 出来事] + [シリーズマーカー] + 【メタ情報】
```

**理想構成例**:
```
【○○!?まとめ】Aと Bの「○○」エピソード総集編①【Aの名/Bの名/にじさんじ】
```

要件:
- フックブラケット: 感情ワード + ?/!
- ペア感を「と」「×」で明示
- 鉤括弧「」で見どころを引用強調
- ① マーカーでシリーズ化を示唆
- 文字数: **45〜55 字** が安全圏

---

## 4. 禁則

| 禁則 | 理由 |
|---|---|
| 絵文字（長尺動画） | -11% 視聴（VidIQ） |
| 60 字超 | UI truncate |
| 「まとめ」「総集編」を同タイトル内で重複 | 冗長感 |
| 数字の羅列 (例: 100万再生達成記念！) | CTR 弱い傾向 |
| 過剰な大文字英数字 | spammy 認定リスク |

---

## 5. 提案の出し方

新規動画のタイトル提案を受けたら:

1. **元タイトルを 5 つの観点で診断**
   - フック強度（句読点・感情ワード）
   - ペア感（「と」「×」「キャラ名引用」）
   - シリーズマーカー（①②）
   - 文字数（60 字以内）
   - 禁則チェック（絵文字・冗長表現）

2. **3 案出す**
   - A: 軽微調整版（既存スタイル踏襲、リスク最小）
   - B: フック強化版（中庸）
   - C: ミーム最大化版（攻め）

3. **推奨を明示**
   - 通常は A 案
   - 攻めるなら C
   - B は中間

4. **絵文字は使わない**（前回の誤誘導訂正後の規則）
