## 2. H1: 最初の slot のタイミングは「avg 視聴の 60%」が中央値

### 観測

| 統計量 | first_slot_sec / avg_watch_sec |
|---|---|
| 25 パーセンタイル | 0.25 |
| **中央値** | **0.60** |
| 75 パーセンタイル | 0.75 |

つまり**動画の半分以上は、最初の mid-roll を「平均視聴時間の 6 割」を過ぎてから配置している**。

### 二極分化

| 集団 | n | 代表動画 |
|---|---|---|
| 早期配置（avg×0.3 以下） | 6 / 19 | `4ZYJdZRvDwo` (0.13), `bP_Vn19u6Y4` (0.13), `27WNi57l-L8` (0.17), `tiGERav_1iQ` (0.18, auto), `aZ-PiV22CFs` (0.19) |
| 中央付近 | 8 / 19 | — |
| 過剰遅延（avg×0.8 以上） | 5 / 19 | `sPtG_pnU-xw` (0.83), `L5nY1arYJ1M` (0.97), `8zzMPgZ6bC8` (1.01), `yeahBDJu-Xg` (1.25), `bv2iwq17LVY` (1.32) |

### 仮説 H1

> **「動画長の前半 1/3 〜 中盤の境目」を直感で 1 本目の配置基準にしている可能性が高い。**
> その結果、平均視聴時間が短い（compilation/総集編の典型: 14〜17 分）動画群で、
> 1 本目が avg 視聴を過ぎる致命位置に集中する。

根拠:
- 過剰遅延 5 本のうち 4 本（`bv2iwq17LVY`, `8zzMPgZ6bC8`, `L5nY1arYJ1M`,
  `sPtG_pnU-xw`）の **first_slot_sec / length_sec ≒ 0.30〜0.36**。
  これは「動画の 30% 地点」というルール of thumb で配置されたとすれば説明可能。
- 早期配置 5 本では `first_slot_sec / length_sec ≒ 0.04〜0.07`。意図的に「冒頭直後」
  に置くルール下にあると見られ、明確に方針が分岐している。

## 3. H2: ゴールデンゾーンへの slot 配置は 30% にとどまる

### 観測

| 指標 | 値 |
|---|---|
| ゴールデンゾーン内 slot 数 / 有効 slot 総数 | 25 / 82 = **30.5%** |
| 動画あたりのゴールデン slot 中央値 | **1.0 本** |
| 動画あたりの有効 slot 中央値 | 4.0 本 |
| ゴールデンに 1 本も置いていない動画 | 3 / 19 (16%) |

### 「ゴールデン slot ゼロ」3 動画

| video_id | 長さ | avg視聴 | 有効 slot | 1本目位置 |
|---|---|---|---|---|
| `bv2iwq17LVY` フレンとこ① | 60.7分 | 16.8分 | 4 | 22:15 |
| `8zzMPgZ6bC8` 30万ガチャ | 47.5分 | 14.2分 | 2 | 14:21 |
| `yeahBDJu-Xg` ベルアン捏造① | 22.4分 | 7.9分 | 2 | 9:52 |

これら 3 本の合計収益は **¥1,338,k（19 動画合計の 47%）**。最も価値の高い動画
群がゴールデンゾーンを完全に空けているのは、収益最適化の観点で最も大きな
損失要因。

### 仮説 H2

> **「視聴者が定着する初期 5 分には広告を入れない」という UX 配慮が
> 業務ルールとして埋め込まれており、結果として収益機会の 70% を逃している。**

根拠:
- 早期配置 5 本のうち auto 1 本を除く 4 本（manual）は明確に「最初の 2 分以内」
  に 1 本目を配置している。一方、過剰遅延 5 本は全て manual。
  → manual 内で**作業者・時期・動画タイプ**による方針差がある可能性。
- ゴールデンゾーン内 slot 比率と RPM の関係を別途確認したい。本チャンクでは
  Spearman 相関を計算していないが、Chunk 4 のモデル乖離分析で間接検証する。

## 4. H3: 自動挿入は配置密度で勝る、ただし RPM では大きく劣る

### 観測

| 指標 | manual (n=17) | auto (n=2) | 倍率 |
|---|---|---|---|
| 中央 first_slot / avg_watch | 0.62 | **0.27** | 0.43x |
| 中央ゴールデン slot 数 | 1.0 | **2.0** | 2.0x |
| 有効 slot / 動画長(分) | 0.08 | **0.33** | 4.1x |
| **中央 RPM (¥)** | **556** | **159** | **0.29x** |

### 自動挿入の 2 動画

| video_id | 長さ | avg視聴 | slot 数 | 1本目 | RPM | 収益 |
|---|---|---|---|---|---|---|
| `tiGERav_1iQ` (アンマリ①) | 15.1分 | 5.6分 | 5 | 1:00 | ¥67 | ¥10,567 |
| `6zA8ajECkDA` (ツボおじ) | 18.8分 | 7.5分 | 6 | 2:36 | ¥251 | ¥17,250 |

### 仮説 H3

> **YouTube の自動挿入は配置原則として優れている**（早期配置、密度、ゴールデン
> ゾーンへの厚い配分）**が、これらの動画は元々 RPM が低い特性を持っており、
> 単純比較で結論を出すのは危険**。

根拠と注意:
- auto 2 本のうち `tiGERav_1iQ` はホロライブ切り抜きを含み、収益化制限の
  可能性が高い（cleaned.csv の RPM 効率分析で Bottom 5 入り）。
  → RPM 差の主因は「コンテンツ要因」で、配置方式の優劣を直接示すものではない。
- とはいえ、配置原則の比較指標（first_slot, slots/min, ゴールデン slot 数）は
  全て auto > manual。**「auto を雛形に手動配置の原則を見直す」価値がある**
  と仮説する。
- 反例検証として、上位 manual 動画（`27WNi57l-L8`, `IfqMm5O2yKI`,
  `4ZYJdZRvDwo` など）の first_slot は 0.13〜0.32 で、auto と同水準。
  → manual の中にも好例が混在しており、「manual だから悪い」ではなく
  **「manual 内のばらつきが大きい」**のが本質。

## 6. H5: モデル乖離は「slot 密度」と強く逆相関する

### 観測

`実測広告/セッション − モデル予測値` を **model_gap** と定義。

| 統計量 | model_gap |
|---|---|
| 中央値 | -0.16 |
| 平均 | -0.49 |
| 25 / 75 パーセンタイル | -1.06 / +0.06 |
| min / max | -2.18 / +0.81 |

### 決定的なパターン: slot 密度との関係

| slots/分 ランク | 動画 | slots_per_min | model_gap |
|---|---|---|---|
| 上位 5 | `tiGERav_1iQ` (auto) | 0.33 | -1.55 |
| | `6zA8ajECkDA` (auto) | 0.32 | -1.72 |
| | `4ZYJdZRvDwo` | 0.25 | **-2.18** |
| | `aZ-PiV22CFs` | 0.20 | -1.22 |
| | `bP_Vn19u6Y4` | 0.17 | -1.52 |
| 下位 5 | `bv2iwq17LVY` | 0.07 | **+0.66** |
| | `sPtG_pnU-xw` | 0.07 | +0.81 |
| | `gufD7_XysaE` | 0.06 | +0.08 |
| | `ROzfh_3D5Jo` | 0.06 | +0.19 |
| | `XsefWP-oKek` | 0.05 | -0.48 |

**Spearman 相関 ρ(slots_per_min, model_gap) = -0.81**（Pearson -0.86）。

### 仮説 H5

> **slot 密度を上げても、線形に広告表示が増えるわけではない。**
> 高密度ゾーンでは「YouTube 内部のペーシング規則 or 視聴者の早期離脱」によって
> 想定の半分〜3分の1しか impression が生まれていない。

含意:
- **「とにかく slot を増やせばいい」は誤り。**
  `4ZYJdZRvDwo`（8 slots, 31.5分）は実測 1.92 ads/session に留まる。
  本来モデル上は 4.10 ads/session 出るはずが、半分以下しか機能していない。
- 一方、低密度（0.07 slots/分以下）の動画は実測 > モデル。これは
  **avg_watch を超えた位置の slot でも、ロングテール視聴者が想定より多く
  到達している**ことを示唆。retention curve の右端を引き上げる必要あり。
- 実務上の含意: **slots/min の天井は概ね 0.15 程度**（密度 0.15 を超えると
  model_gap が急速に負化する閾値が見える）。配置の上限ガイドラインとして
  「動画長(分) × 0.15」を slot 数の上限と仮置きできる。

### 検証すべきこと

1. ad_impressions の内訳（pre-roll / mid-roll / post-roll）が分離できれば、
   高密度動画で特に何が削られているか確定できる。
2. 視聴維持率カーブの実測値が手に入れば、retention curve の校正で
   model_gap を 0 付近に揃えられる。
